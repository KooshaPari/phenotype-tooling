# Tool Catalog

## Quality And Docs

- [`quality-gate`](https://github.com/KooshaPari/phenotype-tooling/tree/main/crates/quality-gate)
  aggregates common Rust checks into one pass/fail binary.
- [`docs-health`](https://github.com/KooshaPari/phenotype-tooling/tree/main/crates/docs-health)
  validates documentation health.
- [`doc-link-check`](https://github.com/KooshaPari/phenotype-tooling/tree/main/crates/doc-link-check)
  checks Markdown links without a broad crawler.

## Traceability

- [`fr-trace`](https://github.com/KooshaPari/phenotype-tooling/tree/main/crates/fr-trace)
  scans FR identifiers and test references.
- [`fr-coverage`](https://github.com/KooshaPari/phenotype-tooling/tree/main/crates/fr-coverage)
  reports requirement coverage.
- [`temporal-grounding`](https://github.com/KooshaPari/phenotype-tooling/tree/main/crates/temporal-grounding)
  provides time and elapsed-duration helpers for agents.

## Release And Governance

- [`release-cut`](https://github.com/KooshaPari/phenotype-tooling/tree/main/crates/release-cut)
  plans semantic release changes.
- [`commit-msg-check`](https://github.com/KooshaPari/phenotype-tooling/tree/main/crates/commit-msg-check)
  validates commit messages.
- [`legacy-scan`](https://github.com/KooshaPari/phenotype-tooling/tree/main/crates/legacy-scan)
  detects legacy scripting anti-patterns.
- [`sbom-gen`](https://github.com/KooshaPari/phenotype-tooling/tree/main/crates/sbom-gen)
  generates software bill-of-materials output.

## Agent Operations

- [`agent-orchestrator`](https://github.com/KooshaPari/phenotype-tooling/tree/main/crates/agent-orchestrator)
  coordinates multi-lane agent execution.
- [`agent-forecast`](https://github.com/KooshaPari/phenotype-tooling/tree/main/crates/agent-forecast)
  estimates agent-work progress and risk.
- [`anthropic-usage-poll`](https://github.com/KooshaPari/phenotype-tooling/tree/main/crates/anthropic-usage-poll)
  polls usage data for operational reporting.
